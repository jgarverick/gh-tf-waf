# Contributing to the GitHub Well-Architected Framework Implementation

We welcome contributions to the GitHub Well-Architected Framework Implementation!

## How to Contribute

1.  **Fork the repository.**
2.  **Create a new branch** for your feature or bug fix.
3.  **Follow the coding style** used in the project.
4.  **Write clear commit messages.**
5.  **Test your changes** thoroughly.
6.  **Submit a pull request** with a clear description of your changes.

## Coding Style

*   Use Terraform best practices.
*   Follow the naming conventions used in the project.
*   Write clear and concise code.
*   Document your code with comments where necessary.

## Commit Messages

*   Use a clear and descriptive subject line.
*   Provide a more detailed explanation in the body of the message.
*   Reference any related issues or pull requests.

## Testing

*   Write unit tests for your code.
*   Test your changes in a development environment.
*   Ensure that your changes do not break existing functionality.

## Pull Requests

*   Submit pull requests to the `main` branch.
*   Provide a clear description of your changes.
*   Reference any related issues or pull requests.
*   Be prepared to address feedback from reviewers.
